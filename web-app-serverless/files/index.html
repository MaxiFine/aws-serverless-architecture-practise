Quick start

  1. Edit files under web-ec2-scenario/files/ (e.g., index.html, items.html).
  2. Upload to the provisioned frontend S3 bucket and wait for CloudFront propagation.
  3. Open Items to test the backend API at /api.
  4. See About for architecture and deployment steps.

How it works

Frontend

Static site in S3, served by CloudFront with an Origin Access Control (OAC).

Backend

EC2 Auto Scaling behind an ALB, reachable via /api/* path behavior.

Auth (optional)

Cognito + Lambda@Edge for viewer-request authentication using PKCE.
